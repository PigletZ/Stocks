# 财务分析模块 - 设计文档

> 状态：已实施（2026-07-17 评审确认并落地，见 §9 决策记录）
> 本文档为导航栏「财务分析」模块的设计与实现说明。

## 1. 需求概述

- 导航栏新增「财务分析」模块
- 可选取 1~10 只股票进行分析（**不支持银行股**，选股器直接过滤，见 §3.2）
- 分析内容：三大报表（利润表、资产负债表、现金流量表）
- 时间范围：今年和去年（即 2025 全部四期 + 2026 已披露期）
- 分析呈现：三层视图全做（§4）；只做亮点/风险标签，不做综合评分

## 2. 数据可行性（已实测验证）

2026-07-17 用普通 token 逐项验证，三大报表接口均可用：

| 接口 | 内容 | 实测 | 关键字段 |
|---|---|---|---|
| `income` | 利润表 | ✅ 85 列 | `total_revenue` 营业总收入、`revenue` 营业收入、`n_income_attr_p` 归母净利润、`basic_eps` |
| `balancesheet` | 资产负债表 | ✅ 152 列 | `total_assets`、`total_liab`、`total_hldr_eqy_inc_min_int` 股东权益、`money_cap` 货币资金、`goodwill` 商誉 |
| `cashflow` | 现金流量表 | ✅ 97 列 | `n_cashflow_act` 经营现金流净额、`n_cashflow_inv_act`、`n_cash_flows_fnc_act`、`free_cashflow` |
| `fina_indicator` | 财务指标（预计算） | ✅ | ROE、毛利率、流动比率等（可作派生指标的校验/补充） |

拉取方式：`{api}(ts_code=..., start_date=..., end_date=...)` 按公告日区间一次返回全部报告期。

**数据特性与处理规则**：

1. **同一报告期有多行**（合并报表/母公司报表/更正公告等）：按 `end_date` 分组，优先取 `report_type='1'`（合并报表），再取 `f_ann_date` 最新一行。
2. **报告期覆盖**：以平安银行为例，公告日区间 2025-01-01~今 返回 `20241231 / 20250331 / 20250630 / 20250930 / 20251231 / 20260331` 共 6 期。即"今年和去年"实际对应 **2025 全部四期 + 2026 已披露期**（一季报已出，中报 8 月起陆续披露）。
3. **季报为累计值（YTD）**：Q3 数据是前三季累计。同比必须同口径对齐（2026Q1 vs 2025Q1），不能直接 Q3 减 Q2 当单季（如需单季值需额外计算，本设计暂不做）。
4. **数据变化频率低**：财报只在披露期（4 月底、8 月底、10 月底集中）更新，适合落库缓存。

## 4. 分析结果呈现方案（核心讨论点）

推荐**三层递进**：快速结论 → 横向对比 → 原文追溯。

### ① 财务体检卡（每只股票一张）

最新报告期的核心信息一屏看完：

- 6 个核心指标：营业总收入（+同比）、归母净利润（+同比）、ROE（年化）、毛利率、资产负债率、经营现金流净额
- **亮点 / 风险标签**（规则自动生成，规则见 §5.3），例如：
  - ✅ 亮点：`营收净利双增`、`ROE 连续两期 > 15%`、`现金流充沛`
  - ⚠️ 风险：`增收不增利`、`经营现金流为负`、`资产负债率 > 70%`、`商誉 / 净资产 > 30%`
- 不做综合评分（已确认）

### ② 多股对比视图（≥2 只时）

- **指标对比表**：行 = 指标（按 盈利/成长/偿债/现金流 四组折叠），列 = 股票；每行最优值绿色高亮、最差值红色高亮；支持切换报告期
- **五维雷达图**：盈利能力 / 成长能力 / 偿债能力 / 现金流质量 / 运营效率，每维 0~100 分（相对归一化，见 §5.3），10 只以内不拥挤
- **分组柱状图**：营收规模、归母净利润规模等绝对值指标，X 轴 = 报告期，系列 = 股票

### ③ 报表原文查看器（单股）

- 股票 Tab × 三大报表子 Tab
- 行 = **全部科目**（已确认；展示该股票各报告期非空的全部科目），列 = 报告期（2025 四期 + 2026 已披露期）
- 科目名称用中文字段映射表展示（后端维护 Tushare 字段 → 中文名映射，未映射的显示英文字段名）
- 每列附带同比变化（与去年同期同口径对比）
- 单位统一为「亿元」（保留 2 位小数），支持切换「元」

## 5. 指标体系与计算规则

### 5.1 派生指标（后端统一计算）

| 分组 | 指标 | 公式 | 数据来源 |
|---|---|---|---|
| 盈利 | 毛利率 | (revenue - operate_cost) / revenue | income |
| | 净利率 | n_income_attr_p / total_revenue | income |
| | ROE（摊薄） | n_income_attr_p / 归母权益 | income + balance |
| 成长 | 营收同比 | 本期 total_revenue / 去年同期 - 1 | income |
| | 归母净利同比 | 本期 n_income_attr_p / 去年同期 - 1 | income |
| 偿债 | 资产负债率 | total_liab / total_assets | balance |
| | 流动比率 | total_cur_assets / total_cur_liab | balance |
| 现金流 | 经营现金流净额 | n_cashflow_act | cashflow |
| | 净现比 | n_cashflow_act / n_income_attr_p | cashflow + income |
| 运营 | 存货周转率 | operate_cost / 平均存货（(期初+期末)/2，期初取去年同期期末值） | income + balance |
| | 应收账款周转率 | revenue / 平均应收账款（同上） | income + balance |
| | 总资产周转率 | total_revenue / 平均总资产（同上） | income + balance |
| 风险 | 商誉占净资产比 | goodwill / 归母权益 | balance |

同比口径：同报告期对齐（如 20260331 vs 20250331）；年报对年报。**季报累计值不做单季拆分**（P2 可议）。
周转类指标按当期值计算、不做年化（季报为累计口径，直接可比）。

### 5.2 五维雷达图归一化（对比可视化用，非质量评分）

各维度先在候选股票池内做 Min-Max 归一化（0~100），再取组内平均：

- 盈利能力 = 毛利率、净利率、ROE
- 成长能力 = 营收同比、归母净利同比
- 偿债能力 = 100 - 资产负债率归一值、流动比率
- 现金流质量 = 经营现金流净额、净现比
- 运营效率 = 存货周转率、应收账款周转率、总资产周转率

> 注：相对归一化仅用于**多股对比可视化**（分数只在当次对比的股票间有意义），不是质量评分。体检卡不做评分，只出亮点/风险标签（§5.3）。

### 5.3 体检卡亮点 / 风险标签规则（已确认：不做综合评分）

按最新报告期指标逐条匹配，命中即出标签：

**✅ 亮点标签**

| 标签 | 规则 |
|---|---|
| 营收净利双增 | 营收同比 > 0 且归母净利同比 > 0 |
| 高增长 | 营收同比 > 15% 或归母净利同比 > 15% |
| 高 ROE | ROE（年化）> 15%（连续两期则标注"连续"） |
| 现金流充沛 | 经营现金流净额 > 0 且净现比 ≥ 0.8 |
| 低负债 | 资产负债率 < 40% |

**⚠️ 风险标签**

| 标签 | 规则 |
|---|---|
| 增收不增利 | 营收同比 > 0 且归母净利同比 ≤ 0 |
| 业绩下滑 | 营收同比 < 0 或归母净利同比 < 0 |
| 经营现金流为负 | n_cashflow_act < 0 |
| 利润含金量低 | 净现比 < 0.5（净利为正时） |
| 高负债 | 资产负债率 > 70% |
| 高商誉 | 商誉 / 归母权益 > 30% |
| 毛利率下滑 | 毛利率同比下降 |

阈值初版写死在后端配置中，后续可做成页面可调。

## 6. 后端设计

### 6.1 数据模型（新增 3 张表）

```python
class FinIncome(SQLModel, table=True):
    # 唯一约束: (stock_code, end_date)
    stock_code: str        # 纯数字代码，关联 Stock
    end_date: date         # 报告期
    f_ann_date: date       # 实际公告日（用于判断更新）
    total_revenue: float | None
    revenue: float | None
    operate_cost: float | None
    n_income_attr_p: float | None
    basic_eps: float | None
    data_json: str         # 全科目原文 JSON（供报表查看器）
    updated_at: datetime

class FinBalance(SQLModel, table=True):   # 同上，常用字段列 + data_json
class FinCashflow(SQLModel, table=True):  # 同上，常用字段列 + data_json
```

- 常用字段建列（供派生指标计算）；`data_json` 存该期**全科目**原始 JSON（balancesheet 152 列等），供报表查看器全科目展示
- 写入用 `INSERT ... ON CONFLICT DO UPDATE`（复用 sectordaily 的 upsert 模式）
- 迁移脚本：`backend/scripts/migrate_create_fin_tables.py`

### 6.2 接口

| 接口 | 说明 |
|---|---|
| `POST /api/financial/refresh` `{codes: [...]}` | 对指定股票回源 Tushare 拉取三大报表并 upsert 入库；幂等。前端选完股票后自动调用（只拉缺失/过期数据） |
| `POST /api/financial/compare` `{codes: [...], end_date?: ...}` | 返回每只股票：各期派生指标 + 亮点/风险标签 + 雷达图数据。**前端主接口，一次拿全**（内部先执行 §6.3 的轻量校验） |
| `GET /api/financial/statements?code=&type=income\|balance\|cashflow` | 单股报表原文（按报告期） |

### 6.3 缓存与刷新策略

- **落库缓存**：三大报表数据入库，历史期只增不改
- **每次分析时轻量校验**（替代按天数的被动过期判断）：
  1. 对每只股票调用 1 次 `income(ts_code=...)`（不带日期过滤，返回全部已披露期）
  2. 比对返回的最新 `(end_date, f_ann_date)` 与库内记录
  3. 有新报告期（或更新的公告版本）→ 该股票三表全量回源并 upsert；否则直接读库
- 效果：财报披露当天即可拿到新数据，无需等待；常规情况每股仅 1 次 Tushare 调用
- 调用量估算：分析 10 只股票 ≈ 10 次调用（有新财报的股票额外 +2 次拉 balance/cashflow）

## 10. 亮点 Top100 榜（2026-07-21 确认，实施中）

### 10.1 需求

按亮点规则对全市场股票排名，取 Top 100 展示，入口为财务分析页内 Tab。

### 10.2 约束（实测）

Tushare 财报接口 `ts_code` 必填（`pro.query` 同样校验），**不存在按 period 整批拉全市场的方式**，全市场财报只能逐股回源；业绩快报 `express` 可整批拉但覆盖稀疏不可用。见 docs/tushare_permissions.md「接口参数约束」。

### 10.3 数据流

1. **一次性全量导入**（`scripts/import_fin_all.py`，只跑一次）：对股票池逐股拉三表入库（每股 3 次调用，共约 1.5 万次），限频 + 断点续跑，夜间执行约 1 小时
2. **每日增量**（cron 每交易日 19:00，`scripts/sync_fin_daily.py`）：每股 1 次 income 轻量校验（复用 §6.3），有新公告才三表回源；完成后重算榜单
3. **每日算榜**：纯本地计算（复用 §5 指标 + §5.3 标签），排序取 Top100 落 `finhighlight` 表，支持历史榜单回看

### 10.4 股票池（已确认：与策略选股统一过滤条件 + 银行股）

- 银行股（行业含"银行"，本模块既定规则）
- ST 股票（名称含 ST 或 *ST）
- 退市股票（名称含"退"）
- 科创板（688 开头）
- 创业板（300/301 开头）
- 北交所（8/43/82/83/87/88/89/920 开头）
- 次新股（上市不满 365 天）

复用 `app/strategies/filters.py` 的 `apply_base_filters`。

2. 风险数升序

4. 营收同比降序
